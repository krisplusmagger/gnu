23/*!
24 * \brief A graphical sink to display multiple signals in frequency.
25 * \ingroup instrumentation_blk
26 * \ingroup qtgui_blk
27 *
28 * \details
29 * This is a QT-based graphical sink the takes set of a complex
30 * streams and plots the PSD. Each signal is plotted with a
31 * different color, and the \a set_title and \a set_color
32 * functions can be used to change the label and color for a given
33 * input number.
34 *
35 * The sink supports plotting streaming complex data or
36 * messages. The message port is named "in". The two modes cannot
37 * be used simultaneously, and \p nconnections should be set to 0
38 * when using the message mode. GRC handles this issue by
39 * providing the "Complex Message" type that removes the streaming
40 * port(s).
41 *
42 * This sink can plot messages that contain either uniform vectors
43 * of complex 32 values (pmt::is_c32vector) or PDUs where the data
44 * is a uniform vector of complex 32 values.
45 *
46 * Message Ports:
47 *
48 * - freq (input):
49 * Receives a PMT pair: (intern("freq"), double(frequency)).
50 * This is used to retune the center frequency of the
51 * display's x-axis.
52 *
53 * - bw (input):
54 * Receives a PMT pair: (intern("bw"), double(bandwidth)).
55 * This is used to programmatically change the bandwidth of
56 * of the display's x-axis.
57 *
58 * - freq (output):
59 * Produces a PMT pair with (intern("freq"), double(frequency)).
60 * When a user double-clicks on the display, the block
61 * produces and emits a message containing the frequency of
62 * where on the x-axis the user clicked. This value can be
63 * used by other blocks to update their frequency setting.
64 *
65 * To perform click-to-tune behavior, this output 'freq'
66 * port can be redirected to this block's input 'freq' port
67 * to catch the message and update the center frequency of
68 * the display.
69 */

130 /*!
131 * Set up a trigger for the sink to know when to start
132 * plotting. Useful to isolate events and avoid noise.
133 *
134 * The trigger modes are Free, Auto, Normal, and Tag (see
135 * gr::qtgui::trigger_mode). The first three are like a normal
136 * trigger function. Free means free running with no trigger,
137 * auto will trigger if the trigger event is seen, but will
138 * still plot otherwise, and normal will hold until the trigger
139 * event is observed. The Tag trigger mode allows us to trigger
140 * off a specific stream tag. The tag trigger is based only on
141 * the name of the tag, so when a tag of the given name is seen,
142 * the trigger is activated.
143 *
144 * In auto and normal mode, we look to see if the magnitude of
145 * the any FFT point is over the set level.
146 *
147 * \param mode The trigger_mode: free, auto, normal, or tag.
148 * \param level The magnitude of the trigger even for auto or normal modes.
149 * \param channel Which input channel to use for the trigger events.
150 * \param tag_key The name (as a string) of the tag to trigger off
151 * of if using the tag mode.
152 */
