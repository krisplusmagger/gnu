18static const char* __doc_gr_channels_dynamic_channel_model = R"doc(dynamic channel simulator
19
20This block implements a dynamic channel model simulator that can be used to help evaluate, design, and test various signals, waveforms, and algorithms.
21
22
23This model allows the user to set up an AWGN noise source, a random walk process to simulate carrier frequency drift, a random walk process to simulate sample rate offset drive, and a frequency selective fading channel response that is either Rayleigh or Ricean for a user specified power delay profile.
24
25Constructor Specific Documentation:
26
27Build the dynamic channel simulator.
28
29Args:
30 samp_rate : Input sample rate in Hz
31 sro_std_dev : sample rate drift process standard deviation per sample in Hz
32 sro_max_dev : maximum sample rate offset in Hz
33 cfo_std_dev : carrier frequnecy drift process standard deviation per sample in Hz
34 cfo_max_dev : maximum carrier frequency offset in Hz
35 N : number of sinusoids used in frequency selective fading simulation
36 doppler_freq : maximum doppler frequency used in fading simulation in Hz
37 LOS_model : defines whether the fading model should include a line of site component. LOS->Rician, NLOS->Rayleigh
38 K : Rician K-factor, the ratio of specular to diffuse power in the model
39 delays : A list of fractional sample delays making up the power delay profile
40 mags : A list of magnitudes corresponding to each delay time in the power delay profile
41 ntaps_mpath : The length of the filter to interpolate the power delay profile over. Delays in the PDP must lie between 0 and ntaps_mpath, fractional delays will be sinc-interpolated only to the width of this filter.
42 noise_amp : Specifies the standard deviation of the AWGN process
43 noise_seed : A random number generator seed for the noise source.)doc";
